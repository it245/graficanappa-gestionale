v1.0 MES Completo — Fondamenta, Produzione, Costi Gen-Feb 2026

Sistema gestionale MES completo: dalla gestione ordini all'analisi dei costi, passando per produzione, scheduling, integrazione Onda/Prinect/Fiery e reportistica direzionale.

Architettura & Modello Dati

  • Architettura Laravel 11 con MySQL — deploy su Windows Server / Apache
  • Modello dati: Ordini, OrdineFasi, Operatori, Reparti, FasiCatalogo, CostoReparto, PrinectAttivita, pivot tables
  • Login operatori con codice operatore + login admin protetto da password hashata
  • Middleware AdminAuth — protezione route /admin/*
  • Pagina errori personalizzate (403, 404) e health check endpoint (/health)
  • Fix sessione owner: rimosso doppio middleware web che causava Page Expired (419) al login

Produzione & Operatori

  • Dashboard Owner — vista completa di tutte le fasi attive, ordinate per priorità
  • Dashboard Operatore — fasi filtrate per reparto assegnato
  • Produzione: avvio, pausa, riprendi, termine fase con tracciamento operatore e timestamp
  • Flusso pausa/riprendi completo: creazione record PausaOperatore, bottone Riprendi, auto-chiusura pause a termine fase
  • Ore effettive nette: calcolo automatico lordo meno pause accumulate (secondi_pausa su pivot fase_operatore)
  • Cascata stati fasi automatica (FaseStatoService): quando una fase termina, la successiva diventa “pronta”
  • Editing inline campi ordine/fase via AJAX (qta, note, stato, date, carta...)
  • Dettaglio commessa (owner + operatore) con note prestampa, responsabile, commento produzione
  • Vista fasi terminate con storico
  • Formula priorità: giorni_rimasti − (ore/24) + (ordineFase/10000)
  • Catalogo 96+ fasi di produzione con tempi avviamento e copie/ora
  • Mapping fasi → reparti (stampa offset, digitale, plastificazione, fustella, legatoria, piegaincolla, stampa a caldo, spedizione, esterno...)

Integrazione Onda (ERP)

  • Sincronizzazione automatica con Onda IQ (SQL Server) — import ordini + fasi da ERP
  • Sync schedulata ogni ora (lun-sab 06:00–22:00) + bottone manuale
  • Auto-creazione reparti e fasi catalogo da Onda se mancanti
  • Ricalcolo automatico ore/priorità dopo ogni sync
  • Import campi personalizzati: Note Prestampa, Responsabile, Commento Produzione
  • Import prezzi vendita (TotMerce) e costi materiali (SUM PRDDocRighe.Totale) da Onda
  • Import ordini da Excel (.xlsx) con mapping fasi e deduplicazione
  • Sync Excel bidirezionale: nuove righe e eliminazione righe mancanti
  • Import massivo dal foglio Excel del titolare (foglio “tutto”) con comando artisan import:excel-tutto
  • Scheduler automatico Laravel: sync Excel + Fiery ogni minuto (Windows Task Scheduler)

Integrazione Prinect (Stampa Offset)

  • Dashboard Prinect live con auto-refresh AJAX (polling 10s)
  • Storico attività filtrabile con sync automatica
  • Report per commessa: breakdown per workstep e operatore
  • Correlazione automatica job Prinect ↔ commessa MES
  • Aggiornamento automatico fogli_buoni / fogli_scarto nelle fasi STAMPAXL106
  • Calcolo OEE (disponibilità, performance, qualità) per macchina stampa
  • Alert automatici: macchina ferma, scarti sopra soglia, OEE critico

Integrazione Fiery (Stampa Digitale)

  • Dashboard Fiery V900 live con stato job in tempo reale
  • Sync automatica Fiery → MES: avanzamento fasi stampa digitale (STAMPAINDIGO)
  • Terminazione automatica fasi precedenti al cambio commessa su Fiery

Scheduling & Spedizioni

  • Scheduling Produzione — interfaccia Gantt con 3 viste (per macchina, per commessa, tabella priorità)
  • Turni specifici per reparto (offset 0-23, piegaincolla/fustella 6-22, stampa a caldo 6-22 + sab 6-13, altri 8-17)
  • KPI scheduling: commesse attive, in tempo, in ritardo, critiche
  • Dashboard Spedizione dedicata: Da spedire / Spedite oggi / In attesa
  • Dashboard Spedizione: gestione lavorazioni esterne (reparto esterno) con pulsanti Avvia/Pausa/Termina/Riprendi
  • Forza consegna: chiude forzatamente tutte le fasi e completa la spedizione

Admin & Gestione

  • Dashboard Admin: elenco operatori con codice, ruolo, reparti, stato attivo
  • CRUD operatori da admin: crea, modifica, attiva/disattiva
  • Configurazione tariffe orarie per reparto con storicizzazione (valido_dal / valido_al)

Statistiche, Report & Costi

  • Statistiche operatori: fasi completate, ore lavorate, quantità prodotte con grafici Chart.js
  • Report commessa: riepilogo tempi effettivi vs stimati per ogni fase
  • Lista commesse con filtro completate/in corso + storico archivio
  • Report produzione settimanale stampabile (KPI, top operatori, commesse in ritardo)
  • Report Direzione multi-periodo (settimana/mese/trimestre/semestre/anno): 6 KPI con delta %, colli di bottiglia, trend produzione, performance operatori, analisi pause, scarto Prinect, clienti, commesse in ritardo — 5 grafici + export Excel 5 fogli
  • Report Prinect multi-periodo: produzione offset, scarto %, OEE breakdown, trend, performance operatori, top commesse — 5 grafici + export Excel 3 fogli
  • Report Costi & Margini multi-periodo: calcolo costo effettivo commessa (ore_effettive × tariffa_oraria per fase), costo materiali da Onda, margine vendita, 6 KPI con delta %, top profittevoli / in perdita, costi per reparto — 2 grafici + export Excel 4 fogli
  • Granularità adattiva trend: giorno (settimana/mese), settimana (trimestre/semestre), mese (anno)
  • Export report in PDF (window.print con CSS @media print ottimizzato A4)
  • Dashboard KPI giornaliero owner: fasi completate oggi, ore lavorate, spedizioni
  • Dashboard Owner: menu hamburger laterale per azioni rapide (filtro e stampa sempre visibili)

v2.0 Scheduler Mossa 37 & Ottimizzazione Produzione Apr-Giu 2026

Algoritmo di schedulazione produzione con simulazione a eventi discreti, batching intelligente per minimizzare i tempi di setup, e propagazione automatica delle fasi. In attesa di approvazione per il deploy in produzione.

Scheduler Mossa 37 (pronto, in attesa approvazione)

  • SchedulerService: simulazione a eventi discreti completa (~580 righe PHP)
  • Propagazione: sblocca fasi a stato 0 quando predecessori completati (75% delle fasi = 540 propagate)
  • Batching per fustella: BOBST, JOH, PIEGA, FIN raggruppano per codice fustella (setup ridotto 10 min vs pieno 25 min)
  • Batching XL106: raggruppamento per tipo lavaggio (DRIP OFF, Pantone, Standard)
  • Batching plastificatrice: raggruppamento per tipo plastica + formato carta
  • BOBST blocchi: prima tutti i rilievi, cambio config 1h, poi tutte le fustelle (2-3 cambi vs 40+)
  • Piegaincolla blocchi: PI01/PI02/PI03 con cambio config 1h
  • Piega: ordinamento per cod_art dentro il batch (articoli uguali consecutivi)
  • 11 macchine configurate con turni specifici (XL106 24h, altre 6-22)
  • Parametri avviamento/copie_ora per 50+ fasi
  • Risparmio stimato: 85-93 ore/mese di setup
  • Tempo calcolo: <3 secondi per 900+ fasi
  • Migration: 15 campi scheduler su ordine_fasi
  • PhaseCompleted event + PropagateAvailability listener (propagazione istantanea)
  • Export Excel piano produzione con foglio per macchina
  • Email serale con piano produzione allegato
  • Comando artisan: php artisan scheduler:run [--email] [--to=email]
  • Fallback 3 livelli dashboard: scheduler → fascia urgenza → priorità legacy (pronto, da abilitare)
  • Priorità manuale sempre rispettata nello scheduler (pronto, da abilitare)
  • Cron ogni 15 min + email serale (pronto, da abilitare al deploy)

  • Dashboard sintetica su TV in reparto (kiosk mode, rotazione automatica, 3 pagine)

Magazzino Base & Logistica (da sviluppare)

Gestione magazzino standalone (indipendente da Onda) con scarico materiali tramite lettore QR code USB/Bluetooth.

  • Anagrafica materiali: carta, lastre, inchiostri, fustelle, plastificazione — codice interno, unità di misura, formato, fornitore, soglia minima
  • Movimenti di magazzino: carico (arrivo merce + DDT fornitore), scarico (prelievo per produzione), rettifica inventario con motivazione
  • Scarico tramite QR code: etichette QR generate dal MES, scansione con lettore USB/Bluetooth → campo input identifica materiale, operatore inserisce quantità e commessa
  • Generazione e stampa etichette QR per ogni materiale (QR + nome leggibile + codice)
  • Dashboard magazzino: giacenza attuale, materiali sotto soglia (evidenziati rosso), ultimi movimenti, filtri per categoria/fornitore/periodo
  • Alert soglia minima sul dashboard owner: “X materiali sotto scorta”
  • Report consumo materiali per periodo e per commessa
Hardware consigliato per QR: Lettore Bluetooth portatile (es. Netum C750, ~40€) — si comporta come tastiera, compatibile con qualsiasi PC/tablet senza fotocamera. Consigliati 2-3 unità per coprire magazzino e reparti.

BRT — Analisi DDT Multi-Spedizione (09/03/2026)

Problema

Alcuni DDT (es. 507, 508, 509 di Ferrovie dello Stato) risultavano “In attesa” nella dashboard BRT, ma erano in realtà consegnati. Causa: l'API SOAP BRT GetIdSpedizioneByRMA restituisce esito -22 (risultati multipli) quando un DDT ha più spedizioni/segnacolli con lo stesso riferimento mittente.

Soluzioni implementate

  • Tabella ddt_spedizioni — supporta più DDT per commessa (prima era 1:1 su ordini)
  • Cache tracking — stati BRT salvati nel DB, visibili subito senza chiamate SOAP
  • Ricerca con SPEDIZIONE_ANNO — risolve ambiguità per DDT con storico multi-anno
  • Badge “Multi-spedizione” — viola, distingue DDT con più segnacolli da quelli davvero in attesa
  • Ottimizzazione N+1 — dashboard spedizione da ~250 query a ~5

Limitazione nota

DDT con più spedizioni BRT (esito -22) non sono tracciabili via API senza il segnacollo. L'API GetIdSpedizioneByRMA non supporta risultati multipli. Soluzione futura: campo segnacollo manuale o integrazione con file Excel BRT.

Tablet — Risparmio Batteria

Cause scaricamento rapido

  • Schermo sempre acceso con dashboard in continuo
  • Luminosità alta
  • Browser (Chrome/Edge) aperto H24 — consuma CPU e RAM
  • WiFi debole — il tablet aumenta la potenza radio
  • Batteria degradata (dopo 2-3 anni perdono 30-50% capacità)

Raccomandazioni

  • Attivare modalità risparmio energetico
  • Abbassare luminosità al minimo leggibile
  • Disattivare Bluetooth e GPS
  • Spegnimento schermo dopo 1-2 minuti di inattività
  • Postazioni fisse con cavo di ricarica permanente
