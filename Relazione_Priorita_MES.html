Calcolo Priorita e Precedenza

Dashboard Owner — MES Grafica Nappa S.r.l. — Marzo 2026

1. Formula Priorita

Ogni fase di una commessa ha un valore di priorita calcolato automaticamente dal sistema:

Priorita = giorni_rimasti − (ore_lavoro / 24) + (ordine_fase / 100)

Dove:

  • giorni_rimasti = data prevista consegna − data odierna (negativo se la consegna e scaduta)
  • ore_lavoro = tempo di avviamento + (quantita carta / copie all'ora della fase)
  • ordine_fase = peso della fase nel ciclo produttivo (da 2 a 96, definito in configurazione)
Regola: Piu basso e il numero di priorita, piu urgente e la fase. Le commesse scadute hanno giorni_rimasti negativo, quindi appaiono sempre in cima alla dashboard.

2. Calcolo Ore Lavoro

ore = avviamento + (qta_carta / copie_ora)

Ogni fase ha parametri specifici di avviamento (ore fisse di setup) e copie/ora (velocita di produzione). Esempi:

Ogni fase ha un peso ordinale che riflette la sua posizione nel ciclo produttivo. Questo peso influenza la priorita: a parita di urgenza, le fasi iniziali vengono prima.

4. Ordinamento nella Dashboard

Le fasi nella dashboard owner vengono ordinate per priorita crescente:

  • Valori negativi → commesse scadute, massima urgenza (in cima)
  • Valori bassi positivi → consegne imminenti
  • Valori alti → consegne lontane (in fondo)

A parita di scadenza, le fasi iniziali del ciclo (stampa, peso 2–14) appaiono prima delle fasi finali (spedizione, peso 96).

5. Priorita Manuale

Un operatore o l'import da Excel possono impostare una priorita manuale che sovrascrive il calcolo automatico:

  • Se il campo viene modificato dalla dashboard owner, il flag priorita_manuale si attiva e il ricalcolo automatico non sovrascrive piu il valore
  • Valori nel range da −901 a −999 sono considerati forzature manuali e non vengono mai sovrascritti dal sistema
  • Per riportare una fase al calcolo automatico, si rimuove il flag manuale

6. Esempio Pratico

Commessa con consegna tra 5 giorni, 2000 fogli

Fase STAMPAXL106:

ore = 0.5 + (2000 / 5000) = 0.5 + 0.4 = 0.9 h

Priorita = 5 − (0.9 / 24) + (5 / 100) = 5 − 0.038 + 0.05 = 5.01

Fase FUSTBOBST75X106 (fustella):

ore = 0.5 + (2000 / 3000) = 0.5 + 0.67 = 1.17 h

Priorita = 5 − (1.17 / 24) + (43 / 100) = 5 − 0.049 + 0.43 = 5.38

Fase BRT1 (spedizione):

ore = 0.5 + (2000 / 10000) = 0.5 + 0.2 = 0.7 h

Priorita = 5 − (0.7 / 24) + (96 / 100) = 5 − 0.029 + 0.96 = 5.93

Risultato nell'ordine della dashboard:
1. STAMPAXL106 (5.01) → 2. FUSTBOBST (5.38) → 3. BRT1 (5.93)

Commessa scaduta da 2 giorni, 1000 fogli

Fase STAMPAXL106:

ore = 0.5 + (1000 / 5000) = 0.7 h

Priorita = −2 − (0.7 / 24) + (5 / 100) = −2 − 0.029 + 0.05 = −1.98

Questa fase appare in cima alla dashboard perche ha priorita negativa.
