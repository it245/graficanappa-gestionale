Avvio giornaliero del sistema SOP-001
Ruoli: Admin, Owner
Frequenza: Giornaliera (inizio turno)
Tempo stimato: 5 minuti
  1. Verificare che il server .60 sia acceso.
    Il server di produzione si trova all'indirizzo 192.168.1.60. Verificare che sia raggiungibile tramite ping o accesso diretto.
  2. Verificare che Apache sia attivo.
    Aprire un browser e accedere a http://192.168.1.60. Se la pagina non si carica, accedere al server e avviare il servizio Apache.
  3. Controllare lo stato di salute del MES.
    Accedere a http://192.168.1.60/health. La risposta deve essere "MES OK". Se il messaggio non appare, verificare i log in storage/logs/laravel.log.
  4. Eseguire la sincronizzazione Onda.
    Dalla dashboard owner, cliccare su "Sincronizza Onda" nella sidebar per importare le nuove commesse registrate nel gestionale Onda.
  5. Verificare la dashboard.
    Controllare che le commesse del giorno siano visibili e che i reparti mostrino le fasi assegnate correttamente.
Se il server non risponde, contattare l'amministratore di rete. Il MES dipende dal database MySQL sul server .60; senza connessione nessuna operazione e' possibile.
Sincronizzazione Onda SOP-002
Ruoli: Owner, Admin, Prestampa
Frequenza: Automatica ogni ora / Manuale a richiesta
Tempo stimato: 1-2 minuti
  1. Accedere alla dashboard owner.
    Dalla sidebar sinistra, individuare il bottone "Sincronizza Onda".
  2. Avviare la sincronizzazione.
    Cliccare il bottone e attendere il messaggio di conferma. La sincronizzazione importa le commesse dal gestionale Onda via protocollo SOAP.
  3. Verificare l'importazione.
    Controllare che le nuove commesse siano visibili nella lista commesse. Le commesse appena importate avranno tutte le fasi nello stato 0 (non iniziata).
  4. Gestire eventuali errori.
    In caso di errore timeout, riprovare dopo qualche secondo. Il timeout e' stato impostato a 120 secondi per gestire sincronizzazioni di grandi volumi.
Per sincronizzare una singola commessa specifica, eseguire sul server: php artisan onda:sync [numero_commessa]
La sincronizzazione automatica avviene ogni ora. La sincronizzazione Excel bidirezionale avviene ogni 2 minuti, quella Fiery ogni minuto.

Gestione fase produttiva (operatore) SOP-003
Ruoli: Operatore
Frequenza: Continuativa durante il turno
Tempo stimato: Variabile
  1. Accedere con il proprio codice operatore.
    Dalla schermata di login, inserire il codice personale assegnato. Il sistema ridireziona automaticamente alla dashboard del proprio reparto.
  2. Identificare la fase piu' urgente.
    Le fasi sono ordinate per priorita' (valore piu' basso = piu' urgente). Le commesse con priorita' manuale sono sempre in cima alla lista. Verificare la colonna "Priorita'" e scegliere la prima fase disponibile.
  3. Avviare la lavorazione.
    Cliccare il pulsante "Avvia" sulla fase selezionata. Lo stato passa da 0/1 (non iniziata/pronta) a 2 (avviata). Il cronometro parte automaticamente.
  4. Gestire eventuali pause.
    In caso di interruzione, cliccare "Pausa" e selezionare il motivo dal menu a tendina (es. cambio lastra, attesa materiale, manutenzione). Il tempo di pausa viene registrato separatamente.
  5. Terminare la fase.
    Al completamento, cliccare "Termina". Inserire la quantita' prodotta e gli eventuali scarti. Lo stato passa a 3 (terminata).
  6. Verificare le prossime fasi.
    La sezione "Prossime commesse" mostra le lavorazioni successive ordinate per urgenza nel proprio reparto. Procedere con la fase successiva.
Non lasciare fasi in stato "avviata" a fine turno. Il sistema rileva automaticamente fasi abbandonate (attive da piu' di 4 ore in un giorno diverso) e le termina con dati potenzialmente incorretti.
Gli operatori del reparto Prestampa vengono rediretti automaticamente alla dashboard prestampa dedicata (/operatore/prestampa).
Spedizione ordine SOP-004
Ruoli: Spedizione
Frequenza: A completamento ordini
Tempo stimato: 5-10 minuti per ordine
  1. Accedere alla dashboard spedizione.
    Verificare la sezione "Da consegnare": contiene le commesse con tutte le fasi in stato 3 (terminata) o superiore.
  2. Selezionare la commessa da spedire.
    Usare la sidebar con anchor per navigare rapidamente tra le sezioni (Da consegnare, Parziali, ecc.).
  3. Confermare la consegna.
    Scegliere tra consegna Totale (tutto l'ordine) o Parziale (quantita' ridotta). Lo stato delle fasi passa a 4 (consegnato).
  4. Generare il DDT.
    Cliccare il pulsante "PDF" per generare il Documento di Trasporto in formato PDF stampabile.
  5. Verificare il tracking BRT.
    Se la spedizione avviene tramite corriere BRT, verificare il numero di tracking nella colonna dedicata. Le spedizioni multi-collo (esito -22) sono escluse dalle notifiche di ritardo.
Le note consegne sono bidirezionali tra owner e spedizione. Utilizzare il campo note per comunicazioni urgenti: il destinatario riceve notifica push e segnale acustico.

Gestione lavorazioni esterne SOP-005
Ruoli: Owner, Admin
Frequenza: Al bisogno
Tempo stimato: 2-3 minuti
  1. Identificare la fase da esternalizzare.
    Dalla dashboard owner, individuare la fase che deve essere inviata a un fornitore esterno (es. plastificazione, stampa a caldo, UV spot).
  2. Impostare come lavorazione esterna.
    Selezionare la fase e indicare "Inviato a: [nome fornitore]". Lo stato cambia a 5 (Esterno). La fase viene marcata con il flag esterno = true e appare con il badge "EXT" viola nella dashboard.
  3. Monitorare lo stato.
    Le fasi esterne sono visibili nella sezione "Lavorazioni Esterne" accessibile dalla sidebar. Qui si puo' visualizzare il fornitore, la data di invio e lo stato corrente.
  4. Registrare il rientro.
    Quando il materiale rientra dal fornitore, accedere alla pagina Esterne e cliccare il bottone "Rientro". La fase torna nello stato 3 (terminata) e il flusso produttivo prosegue.
Le fasi esterne sono escluse dalla dashboard operatore. L'operatore non vedra' fasi marcate come esterne nella propria lista lavorazioni.
Scheduler produzione SOP-006
Ruoli: Owner, Admin
Frequenza: Giornaliera o al bisogno
Tempo stimato: 3-5 minuti
  1. Accedere alla sezione Scheduling.
    Dalla dashboard owner, cliccare su "Scheduling" nella sidebar sinistra per accedere alla vista pianificazione.
  2. Eseguire lo scheduler.
    Sul server, eseguire il comando: php artisan scheduler:run. Lo scheduler calcola le priorita' basandosi su 4 livelli: disponibilita', urgenza reale, affinita' batch e sequenza ciclo.
  3. Verificare il diagramma Gantt.
    Dopo l'esecuzione, il Gantt viene aggiornato con le date pianificate per ogni fase e macchina. Verificare che le assegnazioni siano coerenti con la capacita' produttiva.
  4. Consultare il piano Excel.
    Il piano di produzione aggiornato e' disponibile nella cartella condivisa:
    C:\condivisa\mes\piano_produzione.xlsx
Sequenza fasi standard: Stampa(10) → Digitale(11) → Plastica(20) → Caldo JOH(30) → Plastica Lux(31) → Finitura Dig.(35) → Taglio(37) → Rilievi BOBST(39) → Fustella(40) → Finestratura(100) → Piegaincolla(110) → Legatoria(120) → Allestimento(700) → Spedizione(999).
Colli di bottiglia principali: JOH > BOBST > Piegaincolla. Pianificare di conseguenza, eventualmente con turno notturno per JOH.

Backup giornaliero SOP-007
Ruoli: Admin
Frequenza: Giornaliera (automatica)
Tempo stimato: Automatico / 5 min verifica
  1. Verifica del backup automatico.
    Il database MySQL grafica_nappa viene sottoposto a backup automatico giornaliero tramite cron job sul server .60.
  2. Controllo del file di backup.
    Verificare la presenza del file di backup del giorno corrente. Il formato del nome file e':
    backup_YYYYMMDD.sql (es. backup_20260408.sql).
  3. Esecuzione manuale (se necessario).
    Per eseguire un backup manuale, collegarsi al server ed eseguire:
    mysqldump grafica_nappa > backup_YYYYMMDD.sql
  4. Rotazione dei backup.
    Conservare gli ultimi 7 backup. Eliminare i file piu' vecchi per risparmiare spazio disco. Verificare periodicamente che la rotazione automatica funzioni.
  5. Test di ripristino (mensile).
    Almeno una volta al mese, verificare l'integrita' del backup tentando un ripristino su un database di test.
Il database risiede esclusivamente sul server .60. In caso di guasto al server senza backup, tutti i dati di produzione saranno persi. Assicurarsi che i backup vengano eseguiti e verificati regolarmente.
Gestione turni SOP-008
Ruoli: Admin
Frequenza: Settimanale
Tempo stimato: 10-15 minuti
  1. Accedere alla gestione turni.
    Dalla dashboard admin, selezionare la voce "Turni" nel menu.
  2. Selezionare la settimana.
    Navigare alla settimana da pianificare utilizzando i controlli di navigazione.
  3. Inserire i turni per ogni dipendente.
    Per ciascun giorno della settimana, assegnare il codice turno appropriato:
    Codice Significato Orario
    T Tutto il giorno 06:00 - 22:00
    1 Turno 1 (mattina) 06:00 - 14:00
    2 Turno 2 (pomeriggio) 14:00 - 22:00
    3 Turno 3 (notte) 22:00 - 06:00
    F Ferie -
    R Riposo -
  4. Salvare e verificare.
    Dopo l'inserimento, salvare le modifiche. I turni vengono utilizzati dal sistema per il calcolo delle presenze e per identificare eventuali ritardi tramite integrazione NetTime.
La macchina XL106 opera su 3 turni (24h, lun-ven). Le altre macchine operano normalmente su 2 turni (06:00-22:00, lun-ven). Pianificare i turni di conseguenza.
